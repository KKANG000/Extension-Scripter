# Scripter v1.2.0 업데이트 안내

v1.2.0은 **유저가 바로 체감하는 UI 사용성 개선**에 집중한 업데이트입니다.

## 사용자 체감 변화

### 1. 퀵 버튼(🎬) 사용 경험 개선
- 퀵 버튼이 더 안정적으로 유지되어, 새로고침/재로딩 후에도 중복 표시나 이상 동작이 크게 줄었습니다.
- 퀵 버튼 모드를 `사이드바 열기` / `CUT! 실행`으로 바꿨을 때, 툴팁이 현재 모드에 맞게 정확히 표시됩니다.
- 결과적으로 입력창 옆 퀵 버튼을 더 예측 가능하게 사용할 수 있습니다.

### 2. Sidebar ↔ Center Popup 편집 일관성 강화
- `Current / Next / Caution / Core Principle` 입력값이 두 화면 사이에서 더 안정적으로 동기화됩니다.
- 상태 버튼(`READY / ACTION / MONTAGE`)과 `Auto Action` 체크 상태 반영이 더 일관적입니다.
- 어떤 화면에서 작업하든 "지금 상태가 무엇인지" 더 명확하게 보입니다.

### 3. Center Popup 레이아웃 정돈
- 팝업 내 주요 편집 영역 배치를 정리해, 장면 작성 흐름이 더 자연스럽습니다.
- `Rollback`과 `Queue` 정보 위치가 정돈되어 시선 이동이 줄어듭니다.
- 텍스트 입력 영역 가독성이 개선되어 장면 편집에 집중하기 쉬워졌습니다.

### 4. Scene Queue 조작 반응성 향상
- Queue 항목 추가/삭제/순서 이동 시 화면 반응이 더 즉각적으로 느껴집니다.
- 항목 번호/카운트 표시 정확도가 개선되어, 긴 큐에서도 현재 순서를 파악하기 쉽습니다.

### 5. 재로딩 시 UI 안정성 향상
- 확장을 다시 불러와도 사이드바/팝업/메뉴 항목이 중복으로 쌓이는 문제가 크게 줄었습니다.
- 장시간 세션에서도 UI가 꼬일 가능성을 낮춰, 연속 작업이 더 편해졌습니다.

## v1.1.1 사용자 체크 포인트
- 퀵 버튼을 자주 쓰는 사용자일수록, 이번 버전의 체감 개선이 큽니다.
- Queue를 길게 운영하는 사용자에게 편집 속도와 안정성 측면의 이점이 큽니다.
